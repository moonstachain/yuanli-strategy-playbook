## 第 1 章 · 战略内核总览

核心命题：战略 = 战略任务 → 原力OS → 关键战役 → 战略驾驶舱 的可复用流程。

> 本章读完，你将能回答 为什么说「战略不是灵感，是生产流程」？ 战略内核由哪四个环节组成？它们怎么串成闭环？ 四关在战略内核里分别处在哪个位置？

| 件 1 | 本体定义 · 战略内核是什么 |
| --- | --- |

传统认知里，战略是老板的天赋直觉——可遇不可求、无法复制。原力战略的反共识在于：**战略是可被工程化的生产流程，**有标准件、有装配线、有验收口。这是为什么它能规模化教学、能被 AI 放大。
战略内核由四个环节串成一条流水线：

| 环节 | 定义 |
| --- | --- |
| 战略任务 | 用户/组织当前最该解的那个全局最优问题（输入） |
| 原力OS | 一套可复用的思维框架——四关就是它的核心标准件（加工逻辑） |
| 关键战役 | 把战略落成一场可打赢、可验真的具体战役（执行） |
| 战略驾驶舱 | CEO 视角的仪表盘，盯住战役进度与全局健康（监控） |

| 件 2 | 操作模板 · 四环节闭环 + 四关定位 |
| --- | --- |

> 战略任务（输入：全局最优问题）       │    原力OS（加工：四关标准件）    觉醒 → 独创 → 升维 → 锁定       │    关键战役（执行：可打赢的一仗）       │    战略驾驶舱（监控：CEO 仪表盘）       │    ←———— 拿到结果，反哺下一个战略任务

四关在原力OS里的定位——每关解一个战略子问题，串行不可跳：

| 关 | 解的战略子问题 | 本册章节 |
| --- | --- | --- |
| 一 觉醒 | 战略起点：这个人是谁、能持续做什么 | 第 2 章 |

| 三 升维 | 变现端：怎么把能力变成能跑的系统 | 第 4 章 |
| 四 锁定 | 防御端：怎么守住利润十年 | 第 5 章 |

| 件 3 | 合格性自检 · 内核闭环跑通没 |
| --- | --- |

- 我能说清当前的「战略任务」是哪个全局最优问题
- 我知道现在卡在四关的哪一关
- 我有一场正在打的「关键战役」，且它有 SMART 验收标准
- 我有一个能一眼看到全局健康的「战略驾驶舱」（哪怕只是一张表）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 把战略当灵感 | 等「想通了」才动——战略是流程，先按四关跑，边跑边清晰 |

| 四关并行/跳关 | 四关是串行装配线，跳一关=后面全部装歪 |

**Q　****战略真的能流程化吗？顶级战略不都是天才直觉？**
**A　**天才直觉负责 0→0.1（开创），流程负责 0.1→N（复制）。原力战略不否认直觉的价值，但它把直觉之外、可被标准化的 90% 工程化——这样普通操盘手也能产出 80 分战略，而不是只能仰望天才。

## 第 2 章 · 第一关收编 · 原力觉醒（战略起点=人）

核心命题：战略不从市场起，从人起。三圈交集 + 阴影整合 = 战略地基。

> 本章读完，你将能回答 为什么战略的起点是「人」而不是「市场机会」？ 三圈交集怎么落成战略输入？天才区 vs 卓越区怎么分？ 阴影怎么从战略负债变成战略资产（金色阴影）？

| 件 1 | 本体定义 · 觉醒作为战略起点 |
| --- | --- |

第一册讲过「合力=原型力」。在战略内核里，觉醒的角色是**提供战略输入的源代码：**先搞清这个人的三圈交集（非对称优势×非理性痴迷×非线性回报）和原型，再据此选战场。这就是「目标在人之下」在战略层的落地。

| 三圈 | 作为战略输入的意义 |
| --- | --- |
| 非对称优势 | 决定战略的「不可复制点」在哪——壁垒的源头 |
| 非理性痴迷 | 决定战略能不能跑十年——续航的源头 |
| 非线性回报 | 决定战略能不能放大——规模的源头 |

| 件 2 | 操作模板 · 觉醒三步出战略输入 |
| --- | --- |

#### 2.1　第一步：定位天才区（不是卓越区）

| 区 | 判据 |
| --- | --- |
| 天才区 | 做得好 + 做完能量回流、越做越精神 → 战略该建在这 |
| 卓越区 | 做得好 + 做完被掏空、要靠零食供能 → 长期会耗死，不建战略 |

#### 2.2　第二步：12 原型定主次（详见第一册第 3 章全谱）

定出主+副人格，据此判断战略适配的商业方向（英雄→拓荒/销售，智者→咨询/教育，探索者→AI 创业/新品类……）。

#### 2.3　第三步：3-2-1 阴影整合，把负债炼成资产

> 阴影 → 金色阴影（真实案例） 一位龟毛、挑剔的学员，把「挑剔」这个阴影放到对的位置——去死磕一位中老年国学老师的课程品质。结果该品类占了集团流水 1/3，利润从 18% 涨到 24%。阴影不是要消灭的，是放到对的战场就成金色阴影。

| 件 3 | 合格性自检 · 觉醒输出齐了吗 |
| --- | --- |

- 我能 30 秒说清三圈交集（非对称×非理性×非线性）
- 我知道自己的天才区在哪、和卓越区分得清
- 我的 12 原型主+副已定，并知道适配的商业方向
- 我做过 3-2-1 阴影练习，知道金色阴影方向
- 我能输出一句话原力定位（≤30 字）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 从市场机会起手 | 先看「什么赛道火」再硬塞自己——退回传统商科，丢了不可复制点 |
| 把卓越区当天才区 | 建战略在耗能的能力上，跑几个月就断电 |
| 绕开阴影 | 压抑阴影=埋掉那一面的超能力，错过金色阴影 |

**Q　****觉醒听起来很玄，跟战略到底什么关系？**
**A　**觉醒是战略的输入端。战略不是凭空选赛道，是从「你是谁」推导出「你该打哪场仗」。跳过觉醒直接做模式，等于不知道自己的牌就上桌——短期能跑，长期一定在不属于你的战场上耗死。

| 件 5 | 真源引用 |
| --- | --- |

> 原力是帮用户找到自己的底层源代码，再匹配科学创业方法和当下最高效的赋能工具。 目标在人之下——先有人，再有目标。 — 真源 · 主理人 · 品类复盘 · 2026-06-06

> 一句话记住第 2 章 战略从人起，不从市场起。觉醒输出三件战略输入：不可复制点（非对称）、续航（非理性）、可放大（非线性），外加把阴影炼成金色阴影。

## 第 3 章 · 第二关收编 · 品类独创

核心命题：16 字真言验品类 + 三角定位 + 心智定价。

> 本章读完，你将能回答 品类独创的 16 字真言怎么用来验收一个品类？ 三角定位（甜用户×贵任务×入脑表达）怎么落？ 什么是心智定价？为什么不按成本定价？

| 件 1 | 本体定义 · 品类独创在战略里的角色 |
| --- | --- |

第二册「体」已定品类是「原力战略」。第二关在战略内核里的角色，是把品类**工程化验收**——用 16 字真言确认它真的独创，再用三角定位把它落成可执行的市场坐标。

| 16 字真言 | 验收标准 |
| --- | --- |
| 无法分类 | 用户用旧品类装不下你 |
| 重新定义 | 你重新定义了用户要解决的问题 |
| 自带预算 | 用户带现成预算来，不需教育 |
| 品类独创 | 提到这个需求第一个想到你 |

| 件 2 | 操作模板 · 三角定位 |
| --- | --- |

品类落地 = 三角定位，三个顶点缺一不可（详见第二册第 4–6 章）：

| 顶点 | 问题 | 原力战略的答案 |
| --- | --- | --- |
| 甜用户 Who | 服务谁 | 高敏感 × 专家型 × 基本盘 |
| 贵任务 What | 解什么贵问题 | 觉醒/创富/守富/家庭（落投资账户） |
| 入脑表达 How | 一句话怎么说 | 把「做自己」变成「好生意」 |

#### 2.1　心智定价（不按成本，按心智账户）

> 心智定价四锚点 定价锚向用户的心理账户（投资账户 > 社交 > 情绪 > 功能），不锚向你的成本 同一交付换一个账户表述，定价空间差一个数量级（「AI 补习」几千 vs 「家族 AI 入场券」几万） 价格本身是信号：「你的东西很重要、很贵」——低价反而让甜用户怀疑价值 产品阶梯按交付深度定价（第五册「器」详解：¥1.28万 / 16.8–19.8万 / 百万）

| 件 3 | 合格性自检 · 品类工程化验收 |
| --- | --- |

- 16 字真言四维全过（无法分类/重新定义/自带预算/品类独创）
- 三角定位三顶点都填实（甜用户/贵任务/入脑表达）
- 定价锚向心理账户，不是成本加成
- 30 秒电梯测试通过：5 个不同的人听懂且愿付钱

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 成本加成定价 | 按工时算价=功能账户思维，自动压到最低价位。K6 里成本加成是衰退原子 |
| 三角缺顶点 | 有甜用户没贵任务（不知道解什么贵问题），或有贵任务没入脑表达（说不清），定位都站不住 |
| 怕贵而压价 | 低价让甜用户怀疑价值。价格是信号，该贵就贵 |

**Q　****心智定价会不会就是「卖贵」的托词？**
**A　**不是。心智定价的前提是你真的把问题升维到了高账户、且交付能兑现。如果只是把功能账户的东西标高价，用户体验后会反噬。心智定价=价值升维在先，定价跟上在后——顺序不能反。

| 件 5 | 真源引用 |
| --- | --- |

> 在既有品类干翻第一名胜算极低，不如开辟一个全新品类，你天然就是第一。 用户带着现成的心理预算来——觉醒的预算、创富的预算、守富的预算，本就存在。 — 真源 · 原力创业 · 品类独创

> 一句话记住第 3 章 品类独创 = 16 字真言验收 + 三角定位（甜用户×贵任务×入脑表达）+ 心智定价。定价锚账户不锚成本，价值升维在先、定价跟上在后。

## 第 4 章 · 第三关收编 · 模式升维

核心命题：从「个人能干」到「系统能跑」——三链贯通 + 58 原子 + 闭环测试卡。

> 本章读完，你将能回答 为什么「个人能干」不等于「生意能跑」？升维升的是什么？ 三链路（前/后/财）各管什么？怎么贯通？ 58 商业模式原子是什么？怎么用它组合出新模式？ 闭环验证测试卡的 2 万/4 周/10 倍标准怎么用？

| 件 1 | 本体定义 · 升维 = 从能干到能跑 |
| --- | --- |

个人能干，靠的是你这个人；生意能跑，靠的是**一套不依赖你也能转的系统。**模式升维就是把「你脑子里的能力」升维成「可被复制、可被规模化的商业模式」。
升维的载体是三链路——一个完整商业模式的三条命脉：

| 链路 | 管什么 | 模块 |
| --- | --- | --- |
| 前链路 | 渴望度（用户为什么要） | 用户细分 / 价值主张 / 渠道结构 / 客户关系 |
| 后链路 | 可行性（你凭什么能交付） | 核心资源 / 关键业务 / 伙伴网络 |
| 财链路 | 生存性（怎么活下去赚到钱） | 收入来源 / 定价机制 / 成本结构 |

| 件 2 | 操作模板 · 三链贯通 + 58 原子组合 + 测试卡 |
| --- | --- |

#### 2.1　三链贯通：跑通一个真实订单

> 前链路（渴望）→ 用户想要 → 进来       ↓ 后链路（可行）→ 你能交付 → 做出来       ↓ 财链路（生存）→ 收到钱、有毛利 → 活下来       ↓    一个真实订单的完整闭环 = 升维初步成功

#### 2.2　58 原子：商业模式不是发明，是组合

商业模式不需要凭空发明。过去 300 年的商业史沉淀出 **58 种模式原子**（DNA 片段）——你要做的是**挑选 + 组合**出适配 K6、适配你三圈的新配方。每个原子在 K1–K6 各周期有不同的兴衰（详见第六册「守」的 K6 适配热力图）。

#### 2.3　闭环验证测试卡：2 万 / 4 周 / 10 倍

> MVP 级模式测试标准 成本 ≤ 2 万元：验证一个模式假设的花费上限 周期 ≤ 4 周：拿到验证结果的时间上限 空间 ≥ 10 倍：这个模式跑通后的放大空间下限——不到 10 倍不值得做

| 件 3 | 合格性自检 · 升维跑通没 |
| --- | --- |

- 三链路 9 模块都填实，不是空的
- 跑通了至少 1 个真实订单的获客→交付→收款完整闭环
- 找到至少 1 个 LTV/CAC > 3 的超级单点
- 用 58 原子给自己的模式做过组合分析，知道核心原子是哪几个
- 至少 3 张闭环测试卡跑通（2万/4周/10倍）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 个人英雄主义 | 所有环节靠自己扛=没升维。生意能跑的标志是「没有你也能转」 |
| 发明而非组合 | 想凭空发明全新模式——风险极高。用 58 原子组合，站在 300 年验证过的基因上 |
| 跳过验证就放大 | 模式没跑通一个真实订单就大规模投入，10 倍空间没验证就 all in |
| 单点 < 3 还硬做 | LTV/CAC < 3 的单点，规模化只会放大亏损 |

**Q　****我现在生意能赚钱，但全靠我自己，这算升维了吗？**
**A　**不算。那是「个人能干」，不是「系统能跑」。判据很简单：你休息一个月，生意还能不能转？升维的目标是把你的能力沉淀成系统（SOP、原力大脑、标准交付），让生意不依赖你这个人——这正是第四册「术」交付链路和第五册「器」产品载体要解决的。
**Q　****58 原子去哪找？这本书有吗？**
**A　**58 原子全谱 + 每个原子的 K1–K6 演化，放在第六册「守」和附录卷里（对应底表 T2/T3/T5）。本章你只需先掌握「模式是组合不是发明」这个原则，以及三链贯通和测试卡标准。

## 第 5 章 · 第四关收编 · 壁垒锁定

核心命题：四维护城 16 控点——让对手进不来、追不上、拿不走、离不开。

> 本章读完，你将能回答 为什么模式跑通后必须锁壁垒？不锁会怎样？ 四维护城（虚/实/入/出）各是什么？16 控点怎么分布？ K6 时代新增的「数据飞轮」控点为什么关键？

注意：本章给壁垒的**战略框架**（属第三部「法」）；壁垒的 K6 守富展开、30 法则十年推演、数据飞轮温度，在**第六册「守」**详述。法立框架，守做展开。

| 件 1 | 本体定义 · 壁垒 = 让利润守得住 |
| --- | --- |

模式升维是**最大化创富**，壁垒锁定是**最大化守富。**没有壁垒的好生意，赚到的利润会被竞争迅速抹平。壁垒 = 四个方向的护城河，每个方向 4 个控制点：

| 维 | 让对手 | 4 控点 |
| --- | --- | --- |
| 虚（心智） | 进不来 | 品牌溢价 / 特许经营 / 品类独创 / 文化叙事 |
| 实（物理） | 追不上 | 极致成本 / 资产密度 / 链主控制 / 工艺黑箱 |
| 入（锁定） | 拿不走 | 学习成本 / 数据沉淀 / 生态迭代 / 沉没成本 |
| 出（网络） | 离不开 | 用户网络 / 平台效应 / 双边市场 / 生态飞轮 |

| 件 2 | 操作模板 · 16 控点诊断 + 数据飞轮 |
| --- | --- |

#### 2.1　16 控点三段评分

每个控点按高/中/低三段自评，找出可投资升级的控点：

| 段位 | 判据 |
| --- | --- |
| 高 | 竞品 6 个月追不上 → 真壁垒，持续投资 |
| 中 | 有一定门槛但可被追 → 重点升级对象 |
| 低 | 几乎无门槛 → 要么补强、要么放弃这个方向 |

#### 2.2　K6 新增控点：数据飞轮

> 数据飞轮（K6 时代第 16 控点） 用户越多 → 数据越多 → AI 越聪明 → 下一个用户体验越好 → 用户更多。 这是 AI 原生生意独有的、自我强化的壁垒。原力战略的「原力大脑/专家大脑」越被使用越聪明，正是这个飞轮（详见第五册「器」）。

| 件 3 | 合格性自检 · 壁垒立住没 |
| --- | --- |

- 16 控点至少 8 个达到「高分锚点」
- 至少 6 个控点在持续投资中
- 至少 1 个壁垒达到「竞品 6 个月追不上」标准
- 客户转换成本可量化（流失率 < 5%）
- 能输出一页「我的护城河」（含证据 + 30 法则推演）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 赚到钱就停 | 利润不锁=邀请竞争。创富之后必须立刻锁壁垒 |
| 只靠单一壁垒 | 只有品牌没有锁定/网络——单维壁垒容易被绕过，要四维协同 |
| 忽略数据飞轮 | AI 原生生意不建数据飞轮=放弃 K6 最强壁垒 |

**Q　****我们做战略咨询，壁垒不就是「主理人这个人」吗？这能算壁垒吗？**
**A　**个人 IP 是虚壁垒的一种（品牌溢价），但只靠它=生意离不开人，无法规模化也无法传承。真正稳的壁垒是四维协同：品类独创（虚）+ 原力大脑数据沉淀（入）+ 专家大脑数据飞轮（出）。把「人的壁垒」逐步转化成「系统的壁垒」，这是守富的核心，第六册「守」详谈。

| 件 5 | 真源引用 |
| --- | --- |

> 模式升维是最大化创富，壁垒锁定是最大化守富。 四维护城 16 控制点——让对手进不来、追不上、拿不走、离不开。 — 真源 · 原力创业 · 壁垒锁定

## 第 6 章 · 关键战役：全局最优 + 假设验证 + SMART

核心命题：战略不是一堆任务，是选对那一场能撬动全局的仗，打赢它。

> 本章读完，你将能回答 为什么战略要落成「关键战役」而不是任务清单？ 怎么用「全局最优」从一堆事里选出该打的那一仗？ 假设验证和 SMART 怎么保证战役能被打赢、被验真？

| 件 1 | 本体定义 · 关键战役是什么 |
| --- | --- |

战略落地最大的陷阱，是把它拆成一长串待办事项——结果样样都做、样样不深。关键战役的本体是：**在所有能做的事里，只选那一场能撬动全局最优的仗，集中火力打赢、并能验真。**

| 三要素 | 含义 |
| --- | --- |
| 全局最优 | 在所有候选目标里，选能撬动全局的那一个（不是局部最优） |
| 假设验证 | 把战役建立在明确假设上，用最小代价先验真伪 |
| SMART | 把战役定义成可被验收的具体目标（具体/量化/可达/相关/限期） |

| 件 2 | 操作模板 · 选战役 → 验假设 → SMART 锁定 |
| --- | --- |

#### 2.1　全局最优：N≥3 候选对比

不要直接认定「就做这个」。先列 ≥3 个候选战役，按「对全局的撬动力」排序，选最高的那个。只有一个候选=没做全局最优。

#### 2.2　假设验证：先证伪再投入

每场战役都建立在假设上（「甜用户会为 X 付费」）。先用最小代价（参考升维测试卡：2万/4周）验证假设真伪，再决定要不要 all in。

#### 2.3　SMART 锁定 + 战略驾驶舱

> SMART 五字母（缺一不合格） S 具体：目标是一句能指着说的具体事 M 量化：有数字（如公域 1000→私域转化 100→升班 20） A 可达：踮脚够得到，不是空想 R 相关：直接服务于战略任务，不是旁支 T 限期：有明确截止时间

战役进度挂到**战略驾驶舱**（CEO 仪表盘）上实时盯。年度目标就是由一连串 SMART 战役累积出来的：**公域 1000 获客 → 私域 10% 转化 100 人 × ¥1.28万 → 升班 10% 共 20 人 × ¥19.8万 ≈ 千万。**

| 件 3 | 合格性自检 · 战役选对了吗 |
| --- | --- |

- 我列了 ≥3 个候选战役并按全局撬动力排了序
- 我的战役有明确假设，并设计了最小代价验证方式
- 战役目标 SMART 五字母齐全（缺一不合格）
- 战役进度挂在战略驾驶舱上，能一眼看到全局健康

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 任务清单式蛮干 | 列 30 件事全做=分散火力，一件都打不穿。要选一场关键战役集中打 |
| 局部最优 | 只有 1 个候选就开干，错把局部最优当全局最优 |
| 不验假设就 all in | 把没验真的假设当事实，大投入后才发现假设错了 |
| SMART 缺字母 | 缺量化或限期=战役无法验收，打了等于没打 |

**Q　****怎么判断哪一场是「关键」战役？**
**A　**问一句：打赢这一场，是不是其他很多事都会被带动、或变得不重要？关键战役的特征是「撬动全局」——它是杠杆点，不是工作量最大的那件，而是赢了之后全局松动的那件。比如对原力战略，当前关键战役是「跑通创始人出镜短视频的精准获客」——它一通，转化和交付的水自然来。
**Q　****假设验证和模式升维的测试卡是一回事吗？**
**A　**同源。测试卡（2万/4周/10倍）是验证「模式假设」的工具，关键战役的假设验证用的就是同一套最小代价逻辑。区别是：升维测试卡验「模式能不能跑」，关键战役验「这一仗的核心假设真不真」。

| 件 5 | 真源引用 |
| --- | --- |

> 公域 1000 获客做有效翻译，私域 10% 转化 100 人、客单 1.28 万，再 10% 升班 20 人、客单 19.8 万——SMART 锁死每一环。 — 真源 · 主理人 · OSA 年度目标拆解 · 2026-06-06

> 用户结合自身业务选对应的关键战役，用原力大脑完成落地，最后做成果汇报、由负责人点评打分。 — 真源 · 主理人 · 课程产品规划 · 2026-06-06

> 一句话记住第 6 章 战略落成关键战役：全局最优选仗（≥3候选）+ 假设验证（先证伪）+ SMART 锁定（五字母齐），挂上战略驾驶舱。选撬动全局那一仗，集中打穿。

## 第 7 章 · 元工具底盘：OSA 不跳层 · MECE 究竟完备 · 信息五层萃取

核心命题：支撑整个战略内核的，是三件不可见但决定成败的元工具。

> 本章读完，你将能回答 OSA 三段为什么「不跳层」？跳层为什么是 80% 项目失败的根因？ MECE 究竟完备怎么检验任何一个框架？ 信息五层萃取怎么把碎片信息爬升到全局最优？

| 件 1 | 本体定义 · 三件元工具 |
| --- | --- |

前六章的战略内核能跑得动，靠的是三件底层元工具。它们不直接产出战略，但决定战略质量的天花板。

| 元工具 | 本体 |
| --- | --- |
| OSA | 任务流转最小单元：O 全局最优目标 / S 信息五层萃取（策略）/ A SMART 行动 |
| MECE | 究竟完备：任何框架都要互斥（不重叠）且穷尽（无遗漏） |
| 信息五层萃取 | 认知爬升路径：碎片 → 特征 → 观点 → 洞察 → 全局最优 |

| 件 2 | 操作模板 · 三件元工具怎么用 |
| --- | --- |

#### 2.1　OSA 不跳层（动手前先填一张卡）

> O 目标层：要做什么？为什么是这件不是那件？（全局最优）    │  铁律：必须有完整 O 才能进 S S 策略层：怎么打？≥3 候选方案比成功率    │  铁律：必须有完整 S 才能进 A A 行动层：谁做、何时做、怎么验真（SMART）

核心铁律：**跳层 = 80% 项目失败的根因。**人天性会跳到 A（动手有快感），但不填 O 和 S 就开干，白干率极高。3 个月刻意练习，白干率可降 70%。

#### 2.2　MECE 究竟完备（验任何框架）

> MECE 两条 + 用法 互斥 ME：各项之间不重叠（借势归外部、合力归内部，不交叉）。 穷尽 CE：各项加起来无遗漏（财富归因=外部+内部，没有第三类）。 用法：本书每一层框架（两个不变量、四关、三链路、四维护城）都过 MECE 体检——这是它们能当标准件的资格证。

#### 2.3　信息五层萃取（S 段最容易翻车）

> 碎片信息 → 特征信息 → 观点信息 → 洞察信息 → 全局最优信息 （原始）   （归类）    （判断）    （规律）    （决策依据）

S 段不是「列 3 个方案」那么简单——它的本质是**认知从碎片爬到全局最优。**中间任一层缺失，S 就不合格。直接列方案没爬五层，说明没拿住 S 的本体。

| 件 3 | 合格性自检 · 元工具用到位没 |
| --- | --- |

- 动手前我会先填 OSA 卡，不跳层（O 完整才进 S，S 完整才进 A）
- 我的 O 是从 ≥3 候选里选出的全局最优，不是「我想做的」
- 我的 S 爬过信息五层，不是直接列方案
- 我用 MECE 检验过自己的框架（互斥+穷尽）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 跳到 A | 「终于能动手了」——跳过 O/S 直接执行，是白干的头号原因 |
| O 只有一个候选 | 「我想做这个」当成 O——没有 N≥3 对比就不是全局最优 |
| S 直接列方案 | 不爬信息五层、直接给方案，洞察层缺失=策略浮于表面 |
| 框架不过 MECE | 自创的框架有重叠或遗漏，用它推导战略必然漏判 |

**Q　****OSA、MECE 这些跟战略内核是什么关系？为什么放最后？**
**A　**它们是**底盘**——前六章的每一关、每一场战役，内部都在用 OSA 跑流程、用 MECE 验结构、用五层萃取爬认知。放最后讲，是因为你先理解了战略内核的全貌，才能体会这三件元工具无处不在。掌握了它们，你就能自己生产新的战略模块，而不只是套用现成的四关。
**Q　****信息五层萃取太抽象，有没有最简记法？**
**A　**记住一句：别从碎片直接跳到结论。看到一堆信息（碎片），先归类（特征），再形成判断（观点），再提炼规律（洞察），最后才落成决策（全局最优）。跳过中间几层直接下结论，就是「拍脑袋」——而拍脑袋正是战略最大的敌人。

| 件 5 | 真源引用 |
| --- | --- |

> 必须先有完整 O 才能进 S，必须先有完整 S 才能进 A。跳层是 80% 项目失败的根因。 — 真源 · 原力 OS · OSA 卡铁律

> S 的本质是信息层级跃迁：碎片 → 特征 → 观点 → 洞察 → 全局最优。中间任一层缺失，S 就不合格。 — 真源 · 原力 OS · 信息五层萃取

> 一句话记住第 7 章 三件元工具撑起整个战略内核：OSA 不跳层（O→S→A）、MECE 究竟完备（互斥+穷尽）、信息五层萃取（碎片→全局最优）。掌握它们，你能自己生产战略，不只套用四关。
