## 第 1 章 · 三链路总图：不费力的闭环

核心命题：好生意 = 不费力的获客 × 转化 × 交付，来的都是对的人。

> 本章读完，你将能回答 三链路分别是什么？为什么是「翻译/增信/惊艳」这三个词？ 为什么强调「不费力」？不费力和「努力」矛盾吗？ 三链路怎么串成一个闭环、互相喂养？

| 件 1 | 本体定义 · 三链路三板斧 |
| --- | --- |

一笔生意从陌生人到满意客户，只走三步。每一步的本体可以用一个字概括：

| 环节 | 本体 | 失败=什么 |
| --- | --- | --- |
| 获客 | 翻译 | 用户读不懂你，互相 miss right |
| 转化 | 增信（信任） | 用户不信「你真这样吗」 |
| 交付 | 惊艳（高 NPS） | 没超预期，没好评、没转介绍 |

一句话串起来：**用户读不懂你（获客失败）→ 读懂了但不信你（转化失败）→ 信了但交付没惊艳（复购失败）。**三关层层递进，每一关的本体都不是「使劲推」，而是**不费力。**

| 件 2 | 操作模板 · 不费力闭环图 |
| --- | --- |

> 陌生人   │  获客=翻译（把对的话放到对的人身上） 对的人 进私域   │  转化=增信（松弛沟通，你的东西很重要很贵） 成交 体验款   │  交付=惊艳（峰终定律，自主拿结果+点评） 满意客户 → 复购/升班/转介绍   └──────── 反哺获客（口碑即新流量）

#### 1.1　为什么是「不费力」

> 不费力 ≠ 不努力 「不费力」指的是势能差顺流而下——因为三链路都建在你的非对称优势上，所以毫不费力地吸引、成交、转化。它不是偷懒，是把努力前置到了战略对位（第一~三册），到执行层就顺了。费力地获客、费力地转化，往往是前面战略没对位的信号。

| 件 3 | 合格性自检 · 三链路通了吗 |
| --- | --- |

- 我能说清自己在三链路哪一环最卡（翻译/增信/惊艳）
- 进我私域的，大多是「对的人」，不需要大量筛选
- 我的成交过程是松弛的，不是费力说服的
- 我的交付能产生好评/转介绍（NPS 为正）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 用力推销 | 在转化环节使劲说服=信号错了。对的人不需要费力说服 |
| 只顾获客不管交付 | 前面拼命拉新、后面交付平庸=漏桶，复购和口碑都起不来 |
| 三环割裂 | 获客、转化、交付各做各的，不形成闭环，口碑反哺不了流量 |

**Q　****「不费力」会不会让团队懈怠、不拼了？**
**A　**恰恰相反。不费力是结果，不是起点。它要求你在战略层（前三册）把功夫下足——选对甜用户、定对品类、设计好关键战役。战略对位之后，执行层才不费力。如果执行很费力，不是该更使劲，是该回头看战略哪里没对位。

## 第 2 章 · 获客 = 翻译

核心命题：把对的话，放到对的人身上。心智隔离 + 翻译六要素。

> 本章读完，你将能回答 为什么获客的本质是「翻译」，而不是「流量」？ 什么是「心智隔离」？玉佛寺珠子的例子说明了什么？ 翻译六要素是哪六个？营销四阶段怎么走？ 视频/图文/直播，哪个渠道优先？为什么不做长期直播？

| 件 1 | 本体定义 · 获客为什么是翻译 |
| --- | --- |

获客的本体不是「搞流量」，是**翻译——把对的话，放到对的人身上。**翻译的前提是先知道「给谁翻译」：你不知道对面是法国人还是大学生，扯半天都没用。所以获客先于一切的，是甜用户识别（第二册第 4 章）。
一个关键判据：翻译要**不费力。**费力地翻译往往翻译不明白，也意味着供需没对上——「翻译的前提是供和需匹配，才值得翻译」。

| 件 2 | 操作模板 · 心智隔离 + 营销四阶段 + 六要素 |
| --- | --- |

#### 2.1　心智隔离：换框架，做区隔

> 心智隔离（玉佛寺珠子的例子） 心智隔离 = 把用户原来的低认知框架，换成一个高位的、对用户也有利的新框架，和同质化内容做区隔。就像玉佛寺高僧加持过的珠子，和普通珠子从此不在一个心智货架上。「原力」这个词，就是我们的区隔心智。

#### 2.2　营销四阶段（用户心理旅程）

| 阶段 | 动作 |
| --- | --- |
| ① 认知 | 让对的人第一次读懂你（翻译主张） |
| ② 心智隔离 + 信任加工 | 换高位框架做区隔，公域→私域加工信任 |
| ③ 体验 | 低门槛体验款（¥1.28万），不费力地试水 |
| ④ 实战 | 用框架+AI 拿到真实结果，转化高端产品 |

#### 2.3　翻译六要素（所有内容的统一规范）

| 要素 | 含义 |
| --- | --- |
| 极简 | 一眼看懂，不绕 |
| 权威 | 荣格、康波、段永平等真源背书 |
| 故事 | 康波故事 / 学员故事 / 自己故事 / AI 商科故事 |
| 细节 | 具体实操，不空泛 |
| 利他 | 金句 / 硬核长文 / 实操方法论，先给价值 |
| 便捷 | 一句入脑 / 一键生成，降低行动成本 |

六要素是**所有营销内容的统一规范**——每个环节都符合，传播效果才好。它本身也是一个可复用的思维框架。

#### 2.4　渠道优先级

| 渠道 | 判定 |
| --- | --- |
| 创始人出镜短视频 | 最优——眼见为实，不费力时威力最大 |
| 图文 | 次优——选题库 + 图文版（内容交付负责人承接） |
| 长期直播 | 不做——太耗人，对的用户不需要靠直播轰量（除非品牌型） |

| 件 3 | 合格性自检 · 翻译到位没 |
| --- | --- |

- 我翻译前先识别了「给谁翻译」（甜用户画像清晰）
- 我有一个清晰的心智隔离词/框架（如「原力」），和同质化内容区隔
- 我的内容符合翻译六要素（极简/权威/故事/细节/利他/便捷）
- 我把主力放在创始人出镜短视频，不靠长期直播轰量

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 追流量不做翻译 | 以为获客=买量/铺渠道。不翻译清楚，流量来了也 miss right |
| 认知错配不翻译 | 用户带「学 AI 工作流」的期待来，我们给的是战略框架——不提前翻译必有落差 |
| 费力地翻译 | 费半天劲还讲不明白=供需没对上，该回头看是不是甜用户错了 |
| 沉迷长期直播 | 像涛哥那样每天直播很辛苦=费力。对的用户不需要靠直播轰 |

**Q　****「心智隔离」听起来像话术，是不是有点忽悠？**
**A　**不是话术，是**换框架。**玉佛寺珠子的价值不在材质，在「高僧加持」这个新框架——而这个框架对用户也真实有利（心理价值真实存在）。心智隔离的底线是：新框架必须**对用户也有利、且我们真能兑现，**否则就退化成忽悠，交付环节会反噬。
**Q　****为什么坚决不做长期直播？别人都在播。**
**A　**因为对的用户（高敏感+专家型+基本盘）不需要靠直播轰量来打动。直播耗人极大，是「费力」的获客。我们卖的是 1 万+ 的产品，人情对了短时间就成交，不需要每天直播。除非是纯品牌型曝光，否则长期直播是用错力气。

## 第 3 章 · 转化 = 增信

核心命题：转化的本质是信任，不是说服。松弛 + 精准匹配。

> 本章读完，你将能回答 为什么转化的本质是「增信」而不是「说服」？ 什么是「松弛沟通」？为什么聚焦用户价值、不纠结价格？ 瑞贝卡案例：一句话怎么戳中、完成 ¥12980 转化？

| 件 1 | 本体定义 · 转化为什么是增信 |
| --- | --- |

用户从私域到付费，中间隔着一句潜台词：「大哥，你真这样吗？」 转化的本体就是回答这句话——**建立信任（增信），让用户「果然如此」。**不是靠话术说服，是靠信任水到渠成。
关键心法：**聚焦用户自身价值，不纠结产品价格。**一旦你开始辩护价格，就掉进了用户的「比价框架」；聚焦「你的东西很重要、很贵」，反而戳中。

| 件 2 | 操作模板 · 松弛沟通 + 瑞贝卡范本 |
| --- | --- |

#### 2.1　松弛沟通三句心法

> 松弛沟通（降低决策压力） 聚焦用户、不聚焦产品：「你的东西很重要、很贵」——把焦点放回用户自身价值 三个不费力：「精力不费力、时间不费力、钱不费力，你来个体验怎么了？」 传递松弛状态：我们是松弛的、不缺这一单——松弛本身就是最强的信任信号

#### 2.2　瑞贝卡范本（真实转化拆解）

| 阶段 | 发生了什么 |
| --- | --- |
| 初始 | 瑞贝卡并不那么信，说「你讲的营销知识我都懂、都重复」 |
| 旧心态（不可取） | 急于证明「我和别人不一样」，陷入辩护 |
| 新心态（范本） | 一句话破局：「这个不重要，你的东西很重要、很贵」——焦点从产品移到用户 |
| 松弛收口 | 「精力、时间、钱都不费力，来体验怎么了」——降低决策压力 |
| 结果 | 完成 ¥12980 转化，后续给出很好的好评 |

| 件 3 | 合格性自检 · 增信到位没 |
| --- | --- |

- 我转化时聚焦用户价值，不陷入价格辩护
- 我的沟通状态是松弛的，不缺这一单的姿态
- 我用「三个不费力」降低用户决策压力
- 成交后用户给出好评/愿意转介绍（信任真实建立）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 急于证明自己 | 用户说「我都懂」时急着反驳=掉进比价框架。应聚焦「你的东西很重要很贵」 |
| 紧绷的成交 | 急、缺、推=反向信号。松弛才是信任信号 |
| 一上来卖高价 | 刚认识就卖 100 万=费力。先用 ¥1.28万 体验款让钱不费力 |

**Q　****用户说「这些我都懂」，怎么破？**
**A　**别反驳、别证明。瑞贝卡范本给的答案是一句话：**「这个不重要，你的东西很重要、很贵。」** 把焦点从「我的产品好不好」移到「你（用户）本身的价值和问题」。用户要的不是被说服，是被看见。
**Q　****松弛会不会显得我们不专业、不上心？**
**A　**松弛不是敷衍，是笃定。它传递的信号是「我们不缺这一单、我们对价值有底气」——这恰恰是最高级的专业。紧绷、追单、反复证明，反而暴露不自信。松弛的前提是你真的服务过对的人、有真实结果撑腰。

## 第 4 章 · 交付 = 惊艳

核心命题：交付追求高 NPS——流程标准化 + 超预期。峰终定律 + 自主拿结果 + 点评。

> 本章读完，你将能回答 为什么交付的核心指标是 NPS（净推荐值）而不是满意度？ 峰终定律怎么设计交付的「峰」和「终」？ 为什么是「用户自主完成关键战役 + 导师点评」，而不是替用户做？ 六天课前后分段、分阶段解锁，怎么制造价值落差驱动升班？

| 件 1 | 本体定义 · 交付为什么追 NPS |
| --- | --- |

交付的本体不是「把课讲完」，是**追求高 NPS（净推荐值）——让用户超预期、愿意主动推荐。**两个必要条件：① 流程标准化（可复制、不靠人现场发挥）；② 确实打到用户、「果然如此」的爽感。
一句原话定调：「第一我有流程标准，第二确实他妈能打到你、打了你爽，然后导出 NPS。」

| 件 2 | 操作模板 · 峰终定律 + 自主拿结果 + 分阶段解锁 |
| --- | --- |

#### 2.1　峰终定律：设计「峰」与「终」

> 峰终定律 人对一段体验的记忆，由最高峰和结尾两个点决定，不是平均值。交付设计要刻意制造一个「峰」（如专家大脑的想象力震撼、现场拿到 HTML 战略报告）和一个漂亮的「终」（成果汇报+导师点评打分）。

#### 2.2　核心交付模式：用户自主完成关键战役 + 导师点评

| 要点 | 说明 |
| --- | --- |
| 用户自主完成 | 每个学员基于自身业务选 1 个关键战役，用框架+原力大脑落地——不替用户做 |
| 导师点评打分 | 现场成果汇报，由负责人针对性点评打分（这就是峰终的「终」） |
| 不做前端技能交付 | 不进入不擅长的领域（不教纯 AI 工作流技能），避免拉低交付确定性 |
| 前置说清 | 获客转化阶段就讲清「教什么、不教什么」，避免预期落差 |

#### 2.3　六天课前后分段 + 分阶段解锁（制造价值落差）

| 设计 | 内容 |
| --- | --- |
| 前三天 | 标准化框架（觉醒/独创/模式壁垒），只开放部分，隐藏后三天落地亮点 |
| 后三天 | AI 原生拿结果——不是学新知识，是现场落地（围绕单个原力 OS 模型） |
| 分阶段解锁 | ¥1.28万 体验班解锁 3–6 个关键战役；¥19.8万 年卡解锁全量 32 个，分四年修 |
| 通关地图 | 给高端年卡用户发完整通关地图，展示全部可解锁内容，驱动升班 |

#### 2.4　空档期解法

两次线下课之间有空档期，用户会焦虑。解法：**用户业务环境持续变化 → 不断选新的关键战役 → 持续依托原力 OS + 原力大脑解决 → 长期留在体系内。**「我把你洗脑进了一个框架里，你永远在发现，所以你没离开我。」

| 件 3 | 合格性自检 · 交付惊艳没 |
| --- | --- |

- 我的交付有明确标准流程，不靠现场临场发挥
- 我刻意设计了一个「峰」（震撼点）和一个「终」（漂亮收尾）
- 交付模式是「用户自主完成关键战役 + 导师点评」，不替用户做
- 我在获客转化阶段就讲清了「教什么、不教什么」
- 我有空档期解法，让用户持续留在体系内

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 替用户做 | 包办关键战役=用户没有「自己拿到结果」的爽感，且把自己拖进不擅长的领域 |
| 平均用力没有峰 | 全程平稳=没有记忆峰值，NPS 上不去。要刻意造峰造终 |
| 全盘开放无落差 | 前三天把后三天的亮点全讲了=没有价值落差，升班动力为零 |
| 不前置预期 | 不提前说清教什么，用户带错期待来，交付再好也是落差 |

**Q　****为什么不直接帮用户把战役做完？那样他们不是更爽？**
**A　**短期爽，长期废。替用户做，他没有「我自己用框架拿到结果」的成就感，下次还得靠你——这违背了「系统能跑、不依赖人」的升维目标，也把你拖进不擅长的执行领域。「自主完成+点评」让用户带走的是能力，不是一次性结果，这才有长期留存和复购。
**Q　****前三天不全讲，会不会让用户觉得我们藏着掖着？**
**A　**关键在「前置说清」。提前讲明：前三天是框架、后三天是落地拿结果，这是产品设计不是藏私。用户清楚价值阶梯，反而期待后三天。藏着掖着是骗，价值分层是设计——区别在于有没有提前讲清楚。

> 关键战役采用用户自主完成+导师点评的模式，不做大量前端技能交付，避免进入不擅长的领域；需要在获客转化阶段就提前讲清楚，避免预期落差。 — 真源 · 主理人 · 关键战役服务模式 · 2026-06-06

> 你不停在有业务环境的变化、关键战役的选择——基本上是我把你洗脑进了一个框架里，你永远在发现，所以你没离开我。 — 真源 · 主理人 · 长期留存逻辑 · 2026-06-06

> 一句话记住第 4 章 交付 = 惊艳，追高 NPS：流程标准化 + 超预期，峰终定律造峰造终，用户自主完成关键战役+导师点评。带走能力不是结果，前后分段制造价值落差驱动升班。

## 第 5 章 · 单位经济：公域 1000 → 私域 100 → 升班 20 → 千万

核心命题：三链路转化率 × 客单价 × 升班率，拆出可被 SMART 锁死的千万路径。

> 本章读完，你将能回答 一年 1000 万怎么从三链路拆成可执行的数字？ 每一层的转化率、客单价、人数怎么算？ 产品阶梯按什么分层？为什么不是按「学习阶段」？ 为什么定价是 ¥12980 而不调？

| 件 1 | 本体定义 · 单位经济是什么 |
| --- | --- |

单位经济 = 把三链路的每一环转化率、客单价、人数拆开，算清**一个用户从公域到升班，到底贡献多少钱。**它把「一年 1000 万」这个口号，变成一串可被 SMART 锁死、可被驾驶舱监控的数字。

| 件 2 | 操作模板 · 千万漏斗拆解 |
| --- | --- |

#### 2.1　年度漏斗（2026 年 SMART 目标）

| 层 | 转化 | 人数 × 客单 | 贡献 |
| --- | --- | --- | --- |
| 公域获客 | 有效翻译 | 1000 人 | 流量池 |
| 私域转化 | 10% | 100 人 × ¥1.28万 | ≈ 130 万 |
| 升班 | 10% | 20 人 × ¥19.8万 | ≈ 200+ 万 |
| 合计 | — | — | ≈ 1000 万 |

> 公域 1000 ──10%──→ 私域转化 100 × ¥1.28万 ≈ 130万                         │                        10% 升班                         ↓                    20 × ¥19.8万 ≈ 200+万                         │               夯实基础盘 ≈ 千万，明年翻番

#### 2.2　产品阶梯（按交付深度，不按学习阶段）

> 三品同一套理论，按交付陪伴深度分层 关键认知：不是 ¥1.28万 是小学、¥19.8万 是中学——三品本质是同一套原力 OS 理论，按交付和陪伴深度分层。 ¥12980 体验款：标品，3–6 个关键战役，让钱不费力地试水 ¥16.8–19.8万 年卡：规模化主力，AI 原生个性化战略咨询，全量 32 关键战役，长期复购 ¥100–200万 结果付费：非标定制（FDE 模式），不在当前规模化核心范围

#### 2.3　为什么 ¥12980 不调

| 判断 | 理由 |
| --- | --- |
| 定价科学 | 用户认知为「小贵但值」，正好降低决策压力、让钱不费力 |
| 存量不动 | ¥19.8万 尚未充分测试，先保持现价稳定 |
| 增量测试 | 后续做增量测试，不动存量用户的价格 |

| 件 3 | 合格性自检 · 单位经济算清没 |
| --- | --- |

- 我能说出每一层的转化率、客单价、人数（公域→私域→升班）
- 我的年度目标拆成了 SMART 数字，挂在战略驾驶舱上
- 我的产品阶梯按「交付深度」分层，不是按「学习阶段」
- 我知道当前最该补强的是哪一层转化率（杠杆点）

| 件 4 | 反本能 / 失败模式 |
| --- | --- |

| 反本能 | 解释 |
| --- | --- |
| 按学习阶段分层 | 把 ¥1.28万/¥19.8万 当小学/中学=用户困惑「凭什么升班」。应按交付深度分 |
| 只盯总目标不拆漏斗 | 「今年要 1000 万」不拆成转化率×客单×人数=无法 SMART、无法监控 |
| 乱调价格 | 频繁调价扰乱用户认知。存量稳定、增量测试是纪律 |
| 分层混乱伤高价转化 | 翻译不准、价格区隔不清→用户分层混乱→高价产品转化受损 |

**Q　****用户从 ¥1.28万 到 ¥19.8万 到百万，会不会困惑「凭什么越来越贵」？**
**A　**会——如果你按「学习阶段」分层。解法是讲清：三品是**同一套理论，按交付和陪伴深度分层。**¥1.28万 是标品体验，往上是越来越个性化的陪伴、越来越确定的交付结果，甚至帮你做事（FDE）。用户为「更确定的结果 + 更深的陪伴」付更多钱，逻辑就顺了。
**Q　****1000 人公域怎么来？这一层的转化率怎么提?**
**A　**公域 1000 靠第 2 章的获客=翻译（创始人出镜短视频为主）。当前的关键战役恰恰就是把这一层做起来——获客一通，后面私域转化、升班的水自然来。漏斗哪一层最窄，哪一层就是当期的关键战役（第三册第 6 章）。现阶段最窄、撬动力最大的是获客。
